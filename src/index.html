Preface

tensei.css is a lightweight classless CSS framework that is meant to be used on static websites with a strong focus on text, readability, accessibility, and reducing distractions.

Basic Typography

tensei.css respects user choice of fonts by using the following system font stack

:root {
  --sans-font: sans-serif;
  --mono-font: monospace, monospace;
  --serif-font: serif;
}

tensei.css uses fluid typography to set a base font size of at least 1rem and at most 1.125rem. When the browser default font size is 16px, smartphones and smaller screens should have a font size of approximately 16px to accomodate more characters on each line. When using tablets like the iPad, the font size should increase to approximately 17px. When using a laptop or a desktop, the font size should increase to 18px to utilize the additional screen real estate and resolution.

:root {
  --base-font-size: clamp(1rem, calc(0.923rem + 0.315vw), 1.125rem);
}

Since the base font size is fluid and all other font sizes are relative to the base font size, each and every element in tensei.css has a fluid font size. This conservative use of fluid typography allows tensei.css to meet its objectives while avoiding the unnecessary complexity offered by fluid typography.

The size of the headings has been adjusted to remove the visual representation of the HTML5 document outline algorithm from the <h1> tags. The heading tags use a type scale value of 1.2. This makes the size mostly equivalent to the browser default size for headings. It also makes the <h5> and the <h6> tags have somewhat higher font size than usual to make them more useful. tensei.css doesn't believe in increasing heading font sizes to obnoxiously high sizes. It abhors typical social media style attention seeking clickbait design.

The colors of headings has slightly lesser contrast compared to body text to avoid halation issues

Colors

tensei.css makes judicious use of carefully chosen colors to meet the following objectives

  • offer color palettes for light mode and dark mode
  • don't change colors abruptly (for example, blue to red) on user interaction
  • make primary colors perceptually uniform in CIELAB color space and have similar APCA contrast Lc values
  • avoid halation issues in dark mode colors
  • avoid highly saturated colors to prevent Helmholtz-Kohlrausch effect
  • use distinguishable colors, even for people with color blindness conditions like Protanopia, Deuteranopia, and Tritanopia
  • HTML elements and their states should be distinguishable even without colors in grayscale mode

Hyperlinks

tensei.css uses underlines in hyperlinks to ensure that they're distinguishable, even in the absence of colors in grayscale mode. The underlines have an opacity of 50% which changes to 100% on hover. This prevents increasing `border-bottom` size and introducing subtle layout shift.

Here is a sample hyperlink.

Bulletproof TLS Newsletter Archive

When hyperlinks are active or visited, the changes in color aren't abrupt and similar in hue to the primary hyperlinks colors to avoid unnecessary stimulation and distractions.

Address and Time

tensei.css sets the font-family of <time> and <address> to be monospace to avoid any possible visual ambiguities between characters such as O, 0, 1, I, l.

Nested Lists

The inline padding for the 1st level of a nested list is higher than the padding on all the subsequent levels. tensei.css uses fluid spacing typography to reduce the padding on the left on smaller devices like smartphones to conserve space.

:root {
  --first-level-space: clamp(1.5em, calc(1.16em + 1.7vw), 2.25em);
  --second-level-space: clamp(1em, calc(0.69em + 1.56vw), 1.69em);
}

Accordions

Accordions made using the <details> tag are useful for displaying notices, FAQs, warnings, and other content that isn't directly related to the primary content.

It's advised to use a block element inside <details>, even if it's a <p> tag for a few words.

NOTE

A border has been drawn around <details> to highlight the extent of the text inside it.

Definition Lists

tensei.css applies minimal adjustments to defintion lists to make them accessible.

A list of important directories in neovim

plugin

If needed, a global plugin can be used to extend the functionality of neovim for all types of files. Neovim will load plugins automatically without us needing to perform any Lua require calls in the lua/ folder as people usually do in their neovim config files.

after
after/syntax/

The after directory is read after all the directories in the runtimepath are read. The after/syntax/ will load syntax files after the runtime path syntax files are loaded.

ftdetect
If you want to make neovim recognize a filetype that it doesn't, you can write autocommands inside Lua scripts inside this directory to make neovim set a recognized filetype.

Tables

A table can either be a normal table or a grid table. A normal table is declared using the <table> tag. The border-bottom of <thead> and the border-top of <tfoot> are bigger than the borders of the table cells.

lua (methods) lua (no methods) global local
vim.opt vim.o set set
vim.opt_local vim.bo/vim.wo - set
vim.opt_global vim.go set -

If a grid table is used, alternate rows have different colors. A grid table can be created by using <table role="grid">.

vim.opt vim.o set set
